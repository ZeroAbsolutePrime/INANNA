# Agents

This repository is designed for future human and multi-agent AI builders who need to understand **INANNA NYX** before attempting implementation.

## Repository Role

This is a documentation-first constitutional repository.

Its center is not code execution. Its center is the preservation of:

- purpose
- law
- ontology
- governance
- civic horizon
- first-proof strategy
- inheritance logic

## Required Reading Order

Every agent should read in this order before proposing changes:

1. `README.md`
2. `docs/project_overview.md`

4. `docs/foundational_laws.md`
5. `docs/system_ontology.md`

## Boundary Rules

Agents working in this repository must preserve the following boundaries:

- The constitutional center comes first.
- The older MVP is secondary and must not be rewritten as the whole project.
- This repo should remain clean of mixed implementation residue unless a legacy import is deliberate and documented.
- No implementation proposal should silently overrule the law layer.
- The model layer must never be described as sovereign over governance.

## Expected Agent Roles

A future multi-agent system working on this repository should separate responsibilities clearly.

### Constitutional Interpreter

Preserves the core project sentence, law layer, and non-negotiable truths.

### Ontology And Governance Synthesizer

Maintains conceptual clarity around beings, layers, realms, proposals, and governance.

### First-Proof Strategist

Evaluates candidate first manifestations without confusing them with the constitutional whole.

### Implementation Translator

Converts constitutional and governance material into future technical architectures when capacity exists.

### Archival Steward

Manages any future imports from older repos or notes without letting them distort the active center.

## What Agents Must Not Do

- Treat this repo as if it were already a product backlog.
- Collapse INANNA NYX into a chatbot, shell wrapper, or runtime experiment.
- Confuse the first believable manifestation with the total constitutional horizon.
- Import old mixed-reality material without labeling it explicitly.

## Current Truth

At this phase, the most useful contribution an agent can make is not code generation. It is preserving structural clarity so future implementation can happen without betrayal.
