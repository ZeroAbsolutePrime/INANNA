## 5. Memory With Restraint

Memory is not collection. Memory is selection.

The system must distinguish:

| Memory Type | What It Contains | Who Owns It |
|---|---|---|
| Private memory | Personal to this operator | Operator only |
| Session memory | Active conversation context | Operator |
| Realm memory | This community's shared context | Realm governance |
| Communal memory | Promoted patterns, useful to all | Governed collectively |
| Forbidden memory | Must never be stored | Deleted on detection |

**The Memory Promotion Law:**

A personal insight does not automatically become communal knowledge.
A departmental pattern does not automatically become central authority.
A private vulnerability must not become operational leverage.
A useful pattern can only move upward through explicit governance.

This prevents memory from becoming hidden centralization.

## 8. Data for Harmony, Not Surveillance

Private data in INANNA NYX serves the person who generated it.

It may secondarily serve the community, under governance.
It may never serve the system itself for extraction.
It may never serve external parties without explicit consent.

The distinction:

| Surveillance | Harmony Intelligence |
|---|---|
| Collects behavior to model the person | Learns to serve the person |
| Uses data to influence decisions without consent | Uses data to inform proposals the person approves |
| Builds behavioral profiles for prediction | Builds service profiles for personalization |
| Data flows to the platform owner | Data stays with the operator and community |
| Optimizes engagement (addictive by design) | Supports intention and reflection |
