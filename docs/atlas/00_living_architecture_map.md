## THE SOVEREIGNTY FIELD

This is not a ring. It is a field.
It passes through every ring, every organ, every tool, every connector.

Sovereignty means:
- the operator knows what the system is doing
- the operator can refuse any action
- meaningful change requires proposal and consent
- memory belongs to the person and community, not the machine
- no hidden mutation of system state
- no cloud coordination without transparency
- no behavioral extraction without consent
- local governance precedes network coordination
- the machine cannot override the human covenant

**Sovereignty is not a feature. It is the constitutional substance of the system.**

Without sovereignty passing through all rings,
INANNA NYX becomes an ordinary surveillance platform.

### Mission
Build a sovereign, governed, relational intelligence architecture
that serves human beings, communities, and future civic systems
without domination, hidden extraction, or opaque authority.

### Vision
A world where artificial intelligence is:
- locally owned, locally governed
- constitutionally bounded

- capable of supporting civic harmony without replacing civic decision-making
- coexistent with organic life, not extractive of it

### Constitutional Identity
INANNA NYX is:
- a local-first sovereign intelligence platform
- a governed field of relation between humans and synthetic intelligence
- a proposal-driven, consent-based operating system for AI action
- a multi-body, multi-realm architecture for plural communities
- an open-source platform for communal stewardship

INANNA NYX is not:
- a chatbot or virtual assistant
- a cloud service owned by a company
- an autonomous agent that acts without consent
- a surveillance platform collecting behavioral data
- a tool that optimizes humans without their knowledge

## ARCHITECTURAL PRINCIPLE: DO NOT CLASSIFY BY FEATURE NAME

The most important rule for future builders:

> An app is not an organ.
> A connector is not an intelligence.
> A tool is not a mission.
> A desktop faculty is not the whole head.
> A security perimeter is not a single module.

When building new components, always ask:
**Which ring does this belong to?**

- If it reasons, interprets, remembers, or governs → Inner Organ
- If it is the runtime environment or deployment → Body/OS
- If it reaches into the operator's machine to take action → Sense/Limb
- If it communicates with humans through messaging → Communication
- If it connects to external software the operator uses → App/Realm
- If it connects to remote infrastructure → Connector/Network
- If it protects boundaries and ensures trust → Security Perimeter

## THE ORGAN CARD FORMAT

Every organ, faculty, and component is documented using this format.
Future agents must use this exact format for any new component.

```
# [RING] · [NAME]
## [One-sentence purpose]

### Identity
What it is. What it does. What it must never do.
The name and its meaning.

### Ring
Which architectural ring this belongs to.
Why it belongs there and not elsewhere.

### Correspondences
Python modules and entry points.
What it calls. What calls it.
Data it reads. Data it writes.
Config files. Environment variables.

### Mission
Why it exists. What breaks if it is removed.

### Current State
What works. What is partial. What is placeholder.
Honest assessment. No pretending.

### Limitations
Hardware limitations. Architectural gaps. Known bugs.
What fails on current machine vs DGX Spark.

### Desired Function
What it should do when fully realized.
What it would do with a 70B model on dedicated hardware.

### Dependencies
What it needs to exist.
What it blocks if missing.

### Evaluation
Grade: A / B / C / D / Placeholder
The single most important unresolved problem.
What a future agent should prioritize first.
```
